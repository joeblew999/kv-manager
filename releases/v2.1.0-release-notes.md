# Cloudflare KV Manager v2.1.0

🎨 **Feature Release** - This release introduces visual organization with color tags for namespaces and keys, enhanced metadata support in the Create Key dialog, and improved UI responsiveness for bulk operations.

## 🆕 Highlights

### Namespace Color Tags

Visual color organization for namespaces in Grid and List views:

- **27-color palette** organized in rows by hue (reds, oranges, yellows, greens, blues, purples, neutrals)
- Color picker dropdown with fixed positioning to prevent clipping in table/grid layouts
- Color indicator dot displayed next to namespace icon
- Bottom border stripe on Grid view cards showing selected color
- Optimistic UI updates with rollback on API failure
- New API endpoints: `GET /api/namespaces/colors`, `PUT /api/namespaces/:id/color`
- Database migration version 4 adds `namespace_colors` table
- Full WCAG accessibility: aria-labels, keyboard navigation, focus indicators

### Key Color Tags

Visual color organization for individual keys in the Keys list:

- Same 27-color palette as namespace colors for consistency
- Color picker at start of each key row for quick access
- Optimistic UI updates with automatic rollback on API failure
- New API endpoints: `GET /api/keys/:namespaceId/colors`, `PUT /api/keys/:namespaceId/:keyName/color`
- Database migration version 5 adds `key_colors` table
- Fully accessible with keyboard navigation and screen reader support

### Enhanced Create Key Dialog

Full metadata support when creating new keys:

- **KV Native Metadata (JSON)** field with live validation and formatting via `JsonEditor`
- **Tags section** with inline add/remove functionality (keyboard support: Enter to add)
- **Custom Metadata (JSON)** field for D1-backed searchable metadata
- Tags and custom metadata saved to D1 on key creation for enhanced search capabilities
- Real-time JSON validation with visual feedback
- Full WCAG accessibility: labeled inputs, aria-labels on remove buttons

### Bug Fixes

#### Bulk TTL Update UI Refresh

Fixed Expiration column not updating after bulk TTL update:

- Added `skipCache` parameter to `loadKeys()` function
- After bulk operations complete, keys list now fetches fresh data bypassing API cache
- Expiration timestamps now update immediately without requiring page refresh
- Improves user experience by providing instant visual feedback

## 📥 Upgrade Instructions

### From v2.0.0

1. **Pull latest code:**

   ```bash
   git pull origin main
   npm install
   ```

2. **Deploy and run migrations:**

   ```bash
   npm run deploy
   ```

   Then visit the app - a yellow banner will appear if migrations are needed. Click "Upgrade Now" to apply migrations 4 and 5.

3. **Or apply migrations manually:**
   ```bash
   wrangler d1 execute kv-manager-metadata --remote --file=worker/migrations/apply_all_migrations.sql
   ```

### Docker Users

```bash
docker pull writenotenow/kv-manager:2.1.0
# or
docker pull writenotenow/kv-manager:latest
docker-compose up -d
```

## 🎨 New Features in Detail

### Color Organization System

The new color tagging system provides visual organization at both the namespace and key level:

**Use Cases:**

- Organize namespaces by environment (production, staging, development)
- Group related keys within a namespace by feature or module
- Visually distinguish critical vs. non-critical data
- Create custom organizational schemes that fit your workflow

**Color Palette:**

- 27 carefully selected colors organized by hue
- High contrast for accessibility
- Consistent across namespace and key views
- Colors persist across sessions

### Metadata Workflow Improvements

Creating keys now supports the full metadata system from the start:

**Before v2.1.0:**

- Create key with name and value only
- Edit key to add metadata and tags separately

**After v2.1.0:**

- Create key with name, value, KV native metadata, tags, and custom metadata all at once
- Streamlined workflow reduces steps
- Better for bulk key creation workflows

## 🔄 Database Migrations

This release includes two new migrations:

**Migration 4: `namespace_colors`**

- Adds table for storing namespace color assignments
- Columns: `namespace_id` (TEXT PRIMARY KEY), `color` (TEXT NOT NULL)
- Indexed for fast lookups

**Migration 5: `key_colors`**

- Adds table for storing key color assignments
- Columns: `namespace_id` (TEXT), `key_name` (TEXT), `color` (TEXT NOT NULL)
- Composite primary key on `(namespace_id, key_name)`
- Indexed for efficient queries

Both migrations are applied automatically via the in-app migration banner or can be run manually using the migration script.

## 📋 API Changes

### New Endpoints

**Namespace Colors:**

- `GET /api/namespaces/colors` - Retrieve all namespace color assignments
- `PUT /api/namespaces/:id/color` - Update color for a specific namespace

**Key Colors:**

- `GET /api/keys/:namespaceId/colors` - Retrieve all key colors for a namespace
- `PUT /api/keys/:namespaceId/:keyName/color` - Update color for a specific key

All endpoints include proper authentication, error handling, and audit logging.

## 🐛 Bug Fixes

### Bulk TTL Update UI Refresh

**Problem:** After performing a bulk TTL update operation, the Expiration column in the Keys list did not refresh to show the new expiration times without a manual page refresh.

**Solution:**

- Implemented cache bypass mechanism in the `loadKeys()` function
- Bulk operation completion now triggers a fresh data fetch
- UI updates immediately with new expiration timestamps

**Impact:** Improved user experience and reduced confusion about whether bulk operations succeeded.
