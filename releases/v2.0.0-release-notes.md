# Cloudflare KV Manager v2.0.0

🚀 **Major Release** - This release brings automated database migrations, a metrics dashboard, webhook notifications, enhanced UI controls, and comprehensive TypeScript strict mode compliance.

## 🆕 Highlights

### Automated Database Migrations

In-app database upgrade system with visual banner:

- Yellow upgrade banner appears when schema migrations are pending
- One-click "Upgrade Now" button to apply all pending migrations
- Automatic legacy installation detection for existing deployments
- Schema version tracking via `schema_version` table
- Three initial migrations: initial_schema, job_audit_events, webhooks
- New API endpoints: `GET /api/migrations/status`, `POST /api/migrations/apply`, `POST /api/migrations/mark-legacy`

### KV Metrics Dashboard

View Cloudflare KV analytics and performance data:

- New "Metrics" tab positioned between Search and Job History
- Real-time metrics from Cloudflare's GraphQL Analytics API
- Summary cards showing total operations, reads, writes, and data points
- Operations breakdown with visual progress bars by type
- Average latency table with P50, P90, P99 percentiles by operation type
- Namespace selector and date range presets (Last 7/14/30/90 days)
- 2-minute cache TTL with exponential backoff for rate limiting

### Webhook Notifications

Event-driven HTTP notifications for key operations:

- Full frontend interface for webhook configuration
- Supported events: key_create, key_update, key_delete, bulk operations, backup/restore, job_failed
- Optional HMAC-SHA256 signatures for payload verification
- Test webhooks to verify endpoint connectivity
- New API endpoints: GET/POST/PUT/DELETE `/api/webhooks`, POST `/api/webhooks/:id/test`

### Enhanced UI

- **Rename Key Feature**: Rename keys while preserving value, metadata, and tags
- **Namespace View Toggle**: Grid/List view option with preference persistence
- **Namespace Filter Bar**: Filter namespaces by name on the main page
- **Enhanced JSON Editors**: Live validation, auto-completion, and formatting
- **Audit Log Enhancements**: "All Namespaces" option, additional operation type filters

### Build Optimization

Significant reduction in bundle size:

- Main bundle reduced from 518 KB → 298 KB (-42%)
- Lazy loading for route-level components
- Vendor chunk splitting for better caching
- Inline SVG favicon replacing 2MB placeholder

## ⚠️ Breaking Changes

### Removed: WebSocket Support

- WebSocket infrastructure removed in favor of HTTP polling
- Rationale: Frontend exclusively uses HTTP polling; WebSocket code was unused
- Impact: None for users (feature was already unused)

### Removed: Job Cancellation

- Cancel button and cancellation logic removed from UI and Durable Objects
- Rationale: Feature never worked after migration to HTTP polling, jobs complete too quickly
- Note: Database schema retains `'cancelled'` status for backward compatibility

## 📥 Upgrade Instructions

### From v1.0.0

1. **Pull latest code:**

   ```bash
   git pull origin main
   npm install
   ```

2. **Deploy and run migrations:**

   ```bash
   npm run deploy
   ```

   Then visit the app - a yellow banner will appear if migrations are needed. Click "Upgrade Now".

3. **Or apply migrations manually:**
   ```bash
   wrangler d1 execute kv-manager-metadata --remote --file=worker/migrations/apply_all_migrations.sql
   ```

### Docker Users

```bash
docker pull writenotenow/kv-manager:2.0.0
docker-compose up -d
```
