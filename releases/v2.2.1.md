# Cloudflare KV Manager v2.2.1

🔒 **Security Patch & Dependency Updates** — Addresses the high-severity minimatch ReDoS vulnerability (GHSA-7r86-cg39-jmmj), completes the ESLint 10 migration, upgrades to Node.js 24, and modernizes the CI/CD pipeline.

## 🔒 Security

### GHSA-7r86-cg39-jmmj — minimatch ReDoS (High Severity)

Patched combinatorial backtracking in `matchOne()` with multiple non-adjacent GLOBSTAR segments:

- Updated minimatch override to `^10.2.3` (resolves to 10.2.4), covering all transitive paths via top-level override
- **Docker**: Added minimatch@10.2.4 patch for npm CLI's bundled minimatch in both builder and runtime stages
- Eliminated all npm audit vulnerabilities — `npm audit` now reports **0 vulnerabilities**

### GHSA-3ppc-4f35-3m26 — minimatch ReDoS

- ESLint 10 upgrade eliminated the eslint-chain minimatch vulnerability
- Removed obsolete `brace-expansion` ^2.0.2 override

### CVE-2026-26960 — tar Path Traversal

- Updated tar override 7.5.2 → 7.5.8

### CI/CD Security

- Removed deprecated `fail-on: error` input from `github/codeql-action/analyze@v4`

## 🔄 Changed

### Node.js 24 LTS Baseline

Upgraded from Node 20 to Node 24 LTS across all configurations:

- Dockerfile updated to `node:24-alpine` for both builder and runtime stages
- GitHub Actions workflows updated to Node 24.x
- `package.json` engines field now requires Node.js ≥24.0.0

### ESLint 10 Migration

- Updated `eslint` 9.39.2 → 10.0.2 and `@eslint/js` 9.39.2 → 10.0.1
- Fixed 2 `no-useless-assignment` violations
- Updated `tsconfig.app.json` target/lib from ES2020 → ES2022

### Docker Build Pipeline

- Docker deployments now trigger only on tag pushes (`v*`), not on every push to main
- Prevents duplicate Docker builds during the PR merge → tag release flow

### CodeQL Fix

- Fixed automatic semicolon insertion warning in `handleMigrate` function (`App.tsx:929`)

### ESLint Zero-Suppression Sweep

Eliminated all 12 `eslint-disable` comments through proper code refactoring:

- Extracted CVA variants to separate files (`badge-variants.ts`, `button-variants.ts`)
- Converted inline functions to `useCallback` with proper dependencies
- Replaced `any` types with proper typing throughout
